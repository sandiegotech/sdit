# TECH-101 — Coding & Computational Thinking

You don't have to be a software engineer to benefit from learning to code — but you do have to be willing to think like a machine for a while, which turns out to reveal something interesting about how you think.

## The Central Question

> What happens when you teach a machine to think — and what does that reveal about how you think?

## What This Course Is About

This is an introduction to programming and computational thinking with no prior experience required. You'll learn what algorithms are, how computers execute instructions, and how to write code that does something useful — automating a task, building a simple web page, processing data. But alongside the technical skills, you'll develop a way of thinking: breaking problems into steps, handling edge cases, debugging systematically, and knowing when a computational approach helps and when it doesn't. By the end, you'll have built several small programs and have the foundation to go much further.

## Why It Matters Now

Software is no longer a specialist domain. It's the medium in which modern institutions operate, make decisions, and scale impact. Understanding how software is built — even at a basic level — changes how you see every app, algorithm, and automated system you interact with. And the ability to write even simple code multiplies what you can do on your own.

## By the End of This Course, You Will

- Think computationally: decompose problems, design algorithms, and trace logic step by step
- Write working programs in a beginner-friendly language, using variables, loops, functions, and conditionals
- Work with simple data structures and understand how programs store and process information
- Debug code systematically — reading error messages, forming hypotheses, testing fixes
- Explain, at a conceptual level, how modern AI systems are built and what their limitations are

## The 15-Day Arc

**Week 1 — Thinking**

- Day 1: What is an algorithm? Everyday examples and the logic underneath
- Day 2: Pseudocode — writing instructions precisely enough for a machine before writing code
- Day 3: Logic and conditionals — if/else, boolean values, and branching decisions
- Day 4: Loops and repetition — the core of automation
- Day 5: Functions — reusable building blocks and why they matter

**Week 2 — Building**

- Day 6: Your first real program — a small, working, something-useful piece of software
- Day 7: Data structures — lists, dictionaries, and how programs organize information
- Day 8: APIs — how software talks to other software, and how to use what already exists
- Day 9: A simple web page — HTML, CSS, and a touch of JavaScript
- Day 10: Automating a task — write a script that does something you currently do manually

**Week 3 — Understanding**

- Day 11: How AI models work at a conceptual level — training data, weights, and inference
- Day 12: Your own simple ML experiment — a classifier or predictor built from a tutorial and understood
- Day 13: The ethics of code — what software encodes, who it affects, and who is responsible
- Day 14: What you'd build next — mapping your skills to a project you actually care about
- Day 15: Final project — a small, complete program that solves a real problem, demonstrated and explained

## What You'll Make

- A coding journal documenting your learning process, debugging sessions, and conceptual breakthroughs
- Three short programs completed during the course, each solving a different class of problem
- A final project: one larger program built independently, with written explanation of design choices

## Key Practices

- Read the error message before you ask for help — it usually tells you more than you think
- Debug by forming hypotheses, not by changing random things and hoping something works
- Build the simplest possible version first; complexity is easier to add than to remove
